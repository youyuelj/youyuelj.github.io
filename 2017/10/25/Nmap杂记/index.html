Nmap参数的一些整理

前言

对nmap使用的一些小记录tips

-A : 全面扫描

1、主机发现：

-sP : Ping 扫描
-P0 : 无Ping扫描
-PS : TCP SYN Ping 扫描
-PA : TCP ACK Ping 扫描
-PU: UDP Ping扫描
-PE;-PP;-PM : ICMP Ping Types 扫描
第一个发送ICMP Echo 数据包探测目标，第二个是ICMP时间戳Ping扫描，第三个为ICMP地址掩码Ping扫描
-PR : ARP Ping 扫描
-n : 禁止DNS反向解析
-R : 反向域名解析
–system-dns : 使用系统域名解析器
-sL : 列表扫描
-6 : 扫描 IPv6 地址
–traceroute : 路由跟踪
-PY : SCTP INIT Ping 扫描

2、搜索网络

-T：时序选项 T0-T5不同选项，选项越大扫描越快，牺牲精准度，一般使用T4
-p：端口扫描顺序 可以指定一个端口号或者其范围
-sS：TCP SYN 扫描 相对来说较为隐蔽的扫描方式
-sT：TCP连接扫描
-sU：UDP扫描
-sN;-sF;-sX：隐蔽扫描
-sA：TCP ACK 扫描
-sW：TCP窗口扫描
-sM：TCP Maimon 扫描 此技术与Null，FIN，Xmas扫描完全一样
–scanflags：自定义TCP扫描 六个标志位URG,ACK,PSH,RST,SYN,FIN的任意组合。在非一般场景使用该选项
-sI：空闲扫描 也就是源地址欺骗扫描 -sI + 僵尸主机
-sO：IP协议扫描
-b：FTP Bounce扫描

3、指纹识别与探测

-sV：版本探测 不要过分相信-A与-sV选项，Nmap不能全部躲过一些软件的伪装
–allports：全端口版本探测
–version-intensity：设置扫描强度 可以选择0~9的强度，默认强度为7，强度越高，时间越长，服务越可能被正确识别
–version-light：轻量级扫描 对应强度为2的扫描，两者是等价的
–version-all：重量级扫描 对应强度为9的扫描
–version-trace：获取详细版本信息
-sR：RPC扫描 该选项多用于与其他端口扫描选项相结合使用它对所有发现的TCP/UDP端口执行SumRPC程序NULL命令，来试图确定他们是否为RPC端口
-O：启用操作系统探测
–osscan-limit：对指定的目标进行操作系统探测 该选项仅在使用-O或者-A进行操作系统检测时起作用
–osscan-guess；–fuzzy：推测系统并识别 该选项对无法判断的操作系统进行一个概率取值

4、速率调整

–min-hostgroup：调整并行扫描组的大小 Nmap默认情况下进行扫描的时候，首先开始扫描较小的值，最小为5，随后慢慢增大组的大小，最大为1024，而这个最小值我们可能用此选项进行说明，–max-hostgroup选项用于说明最大值，Nmap不会超过这个大小
–min-parallelism；–max-parallelism：调整探测报文的并行度
–min-rtt-timeout；–max-rtt-timeout；–initial-rtt-timeout：调整探测报文超时 这些选项对于有严格过滤或者不能Ping通的主机有很好的突破效果，一般情况下rtt时间不得小于100ms，也最好不要大于1000ms
–host-timeout：放弃低速目标主机 对一些反应过慢且不重要的主机进行忽略，此选项我们一般设置为1800000ms，不超过半小时，当我们要过滤时，应根据自身的网络情况等多方面因素进行决定
–scan-delay；–max-scan-delay：调整探测报文的时间间隔

5、防火墙/IDS逃逸

-f：报文分段 在Nmap使用-f选项时会将TCP头分段在几个包中，使得包过滤器，IDS以及其他工具的检测更加困难。一些主机会禁止相应的ICMP请求，对于这种情况可以使用报文分段
–mtu：指定偏移大小 使用指定的MTU可以达到逃逸防火墙IDS的目的，需要注意的是偏移量必须为8的倍数
-D：IP欺骗 -D选项可以指定多个IP地址，或者使用RND随机生成几个地址（RND:11 随机生成11个地址），需要注意的是在进行版本检测或者TCP扫描的时候诱饵是无效的。
-sI：源地址欺骗 在之前的空闲扫描中有说到，主要是用作源地址欺骗
–source-port：源端口欺骗
–data-length：指定发包长度 通常情况下，TCP包是40个字节，ICMP Echo有28 个字节，所以在原来的报文基础上随机附加数据可以达到规避防火墙的效果
–randomize-hosts：目标主机随机排序 该选项对目标主机进行随机的排列，最多可达8096个主机，单方面使用这个选项对防火墙/IDS逃逸的效果并不大，如果配合时间选项则会有很好的效果
–spoof-mac：MAC地址欺骗 我们可以使用字符串“0”随机分配一个mac地址，也可以指定一个mac地址，但是最好是真实存在的，才能达到欺骗管理员的效果，–spoof-mac选项可以使用参数0，MAC Address(手动指定一个地址)，Vendor Name（从指定的厂商生成一个MAC地址）。

6、部分script脚本使用

(1)信息搜集

–script ip-geolocation-：IP信息搜集 nmap –script ip-geolocation- www.xxx.com
whois：WHOIS查询 nmap –script whois www.xxx.com，有时候whois查询到的信息并不准确，我们更加热衷于查询whois的历史记录 nmap –script whois –script-args whois whodb=nofollow www.xxx.com
http-email-harvest：搜集E-mail信息 nmap –script http-email-harvest www.xxx.com 耗时较长
hostmap-ip2hosts：IP反查 nmap -sn –script hostmap-ip2hosts www.xxx.com
dns-brute：DNS信息搜索 nmap –script dns-brute www.xxx.com 该脚本的默认线程是5，如果扫描一个大型网络，速度可能会慢，可以设置一下扫描线程 nmap –script dns-brute dns-brute.threads=10 www.xxx.com 如果查询多个域名也可以指定一个列表 nmap –script dns-brute dns-brute.threads=10，dns-brute.hostlist www.xxx.com
membase-http-info：检索系统信息 nmap -p 445 192...* –script membase-http-info 使用该脚本可以轻易的了解目标操作系统的详细信息，配合前面学习的几个参数有更好的效果
smb-security-mode.nse：后台打印机服务漏洞 nmap –script smb-security-mode.nse -p 445 192.xxx.xxx.xxx 使用该脚本可以检测打印机服务漏洞，该脚本我们需要手动指定端口
smb-check-vulns.nse：系统漏洞扫描 nmap –script smb-check-vulns.nse -p 445 192.xxx 进行SMB漏洞扫描，还可以使用U:或T:进行配合
http-stored-xss.nse：扫描web漏洞 nmap -p 80 –script http-stored-xss.nse www.***.com 该脚本可以帮助我们完成发现网站的XSS漏洞，http-sql-injection脚本可以帮助我们发现SQL注入漏洞
snmp-win32-services：通过snmp列举Windows服务/账户 nmap -sU -p 161 –script=snmp-win32-services 192.xxx nmap –sU -p 161 –script=snmp-win32-users 192.xxx 前一个脚本可以获取目标正在运行的服务，后面的脚本可以看到目标所有的账户
dns-brute：枚举DNS服务器的主机名 nmap –script dns-brute –script-args dns-brute.domain=baidu.com 可以使用线程增加速度，使用hostlist指定列表
http-headers/http-sitemap-generator：HTTP信息搜集 1.使用-sV选项即可对HTTP版本进行探测（nmap -sV -p 80 www.xxx.com） 2.使用命令“nmap -p 80 –script=http-headers 目标地址”即可对目标地址进行HTTP头信息探测 3.HTTP目录结构探测 使用命令“nmap -p 80 –script=http-sitemap-generator 目标地址”即可爬行web目录结构
ssl-enum-ciphers：枚举SSL密钥 使用命令“nmap -p 443 –script=ssl-enum-ciphers 目标”即可枚举ssl密钥
ssh-hostkey：SSH服务密钥信息探测 “nmap -p 22 –script ssh-hostkey –script-args ssh_hostkey=full 127.0.0.1”ssh_hostkey的输出格式有四种：full，bubble，visual，all

(2)数据库方面

mysql-databases：Mysql列举数据库 使用命令“nmap -p3306 –script mysql-databases –script-args mysqluser=root，mysqlpass 192.x.x.x”
mysql-variables：列举Mysql变量 使用命令“nmap -p3306 –script=mysql-variables 192.x.x.x”如果端口无法确定的话，可以使用前面学习的扫描进行确认
mysql-empty-password：检查Mysql密码 使用命令“nmap -p3306 –script=mysql-empty-password 192.xxx” 该命令用于检查Mysql是否是空密码，或者是root，或允许匿名登入
mysql-brute：审计Mysql密码 使用命令“nmap -p3306 –script=mysql-brute 192.xxx” 该脚本用于Mysql弱口令，也可以自定义账号密码字典“nmap -p 3306 –script=mysql-brute userdb=路径 passdb=路径 192.xxx”
mysql-audit：审计Mysql安全配置 使用命令“nmap -p 3306 –script mysql-audit –script-args “mysql-audit.username=’数据库账户’，\mysql-audit.password=’数据库密码’，mysql-audit.filename=’?’ ”
oracle-brute：审计Oracle密码 使用命令“nmap –script oracle-brute -p 1521 –script-args oracle-brute.sid=test 192.xxx” oracle-brute脚本用于暴力破解oracle密码，使用-p指向目标端口号，如果需要自定义账号密码字典破解，则“nmap –script oracle-brute -p 1521 –script-args oracle-brute.sid=test –script-args userdb=路径，passdb=路径 192.xxx”
ms-sql-brute：审计Mssql密码 使用命令“nmap -p 1433 –script ms-sql-brute –script-args userdb=路径，passdb=路径 192.xxx”
ms-sql-empty-password：检查mssql空密码 使用命令“nmap -p 1433 –script ms-sql-empty-password 192.xxx”
ms-sql-tables：读取mssql数据 使用命令“nmap –script ms-sql-tables –script-args mssql.username=数据库账户，mssql.password=数据库密码 192.xxx”
ms-sql-xp-cmdshell：mssql执行系统命令 使用命令“nmap –script ms-sql-xp-cmdshell -script-args mssql.username=数据库账户，mssql.password=数据库密码,ms-sql-xp-cmdshell.cmd=“ipconfig” 192.xxx”
pgsql-brute：审计pgsql密码 使用命令“nmap -p 5432 –script pgsql-brute 192.xxx” 

(3)各类暴力破解

http-brute：审计HTTP身份审计 使用命令“nmap –script http-brute -p 80 www.xxx.xxx” 可以对HTTP中账号密码进行简单的破解
ftp-brute：审计ftp服务器 使用命令“nmap –script ftp-brute –script-args userdb=路径，passdb=路径 -p 21 192.xxx”进行暴力破解ftp账号密码
http-wordpress-brute：审计wordpress程序 使用命令“nmap -p80 –script http-wordpress-brute –script-args userdb=路径，passdb=路径 192.xxx”http-wordpress-brute.threading=10，可以设置线程数
http-joomla-brute：审计Joomla程序 使用命令“nmap -script http-joomla-brute 192.*”可以设置–script-args字典和线程
pop3-brute：审计邮件服务器 使用命令“nmap -p110 –script=pop3-brute 192.xxx”
smb-brute.nse：审计SMB口令 使用命令“nmap –script smb-brute.nse -p445 192.*” 对于复杂密码，加上–script-args passdb=路径 自定义字典
vnc-brute：审计vnc服务器 使用命令“nmap –script vnc-brute -p 5900 192.xxx”+字典
smtp-brute：审计SMTP服务器 使用命令“nmap -p 25 –script smtp-brute 192.xxx”即可审计SMTP服务器，通过“nmap -p 25 –script=smtp-enum-users.nse smtp.xxx.com”枚举远程系统所有的用户
stuxnet-detect：检测Stuxnet蠕虫 使用命令“nmap –script stuxnet-detect -p 445 192.*”
snmp-netstat，snmp-processes，snmp-win32-services，snmp-brute：snmp服务安全审计 使用命令“nmap -sU -p 161 –script=snmp-netstat 192.xxx“即可获取目标主机网络连接状态 “snmp -sU -p 161 –script=snmp-processes 192.xxx”通过snmp服务协议枚举运行的系统进程 最后通过“nmap -sU -p 161 –script=snmp-win32-services 192.xxx”获得windows服务器的服务
“snmp -sU -p 161 –script snmp-brute 192.xxx”对snmp服务进行口令审计

7、其他的一些使用

–send-eth：发送以太网数据包 使用命令“nmap –send-eth 192.xxx”
–send-ip：网络层发送 使用命令“nmap –send-ip 192.xxx”
–privileged：假定拥有所有权 使用命令“nmap -privileged 192.xxx”
–interactive：在交互模式中启动 使用命令“nmap –interactive”
-V：查看nmap版本号 使用命令“nmap -V”
-d：设置调试级别 使用命令“nmap -d 1 192.xxx”调试级别为1~9，级别越高数据越多
–packet-trace：跟踪发送接受的报文 该选项会将nmap接收到的报文全部打印出来，使用命令“nmap –packet-trace -p 20-30 192.*”生成少量报文便于分析
–iflist：列举接口和路由 使用命令“nmap –iflist www.xxx.com”
-e：指定网络接口 使用命令“nmap -e eth0 192.xxx”指定eth0网卡发送数据
-oG：继续中断扫描 使用命令“nmap -oG 1.txt -v 192./24”将扫描结果保存为txt文件，然后按下Ctrl+C中断扫描，但有时间需要继续扫描时，使用命令“nmap –resume 1.txt”
firewalk：探测防火墙
vmauthd-brute：vmware认证破解
